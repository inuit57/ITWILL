비밀번호 :

페이지 전달방식 (get / post)

get 방식 / get 메서드

전송할 데이터를(파라미터) URL의 정보에 붙여서 전송
body가 필요없기 때문에 빠른 전송속도
데이터의 길이제약이 존재함 (256byte)
적은 량의 파라미터, 일반적인 데이터를 전달할 때 사용
(* 일반적인 데이터 : 노출되더라도 크게 문제없는 데이터)
파라미터가 주소줄(URL)에 모두 보여진다. => 보안상 취약

post 방식 / post 메서드

전송할 데이터를(파라미터) URL에 전달 X, 본문에 body에 붙여서 전송
body 생성, 많은 량의 데이터를 전달하기 때문에 get방식에 비해서 상대적으로 처리속도가 느리다.
파라미터의 길이 제약이 없음 => 파일 업로드 등에 적합
파라미터가 주소줄(URL)에 전달 X => 보안적으로도 상대적으로 우수
개인정보, 중요데이터를 전달하는 데 사용
